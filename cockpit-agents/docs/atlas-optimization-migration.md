# 阿拓优化总结与四 Agent 迁移记录

日期：2026-06-28

## 1. 阿拓从占位模型到实时角色经历了什么

阿拓的优化不是单次“调几个动作”，而是逐步建立了一套可复制的角色生产与运行时协议。

### 视觉与资产

- 从低质程序化占位模型切换到正式 GLB。
- 自动按包围盒归一化身高、脚底落地和模型中心。
- 修正贴图色彩空间、材质透明、粗糙度、环境反射与阴影。
- 清除隐藏辅助网格对运行时包围盒和导出的干扰。
- 将错误分配给 `LeftEye / RightEye / Jaw / *_end` 的身体权重转移到 `Head`。
- 每个顶点限制为最多四个骨骼影响并重新归一化。
- 建立七个稳定的语义表情通道：
  - `face_blink_soft_L`
  - `face_blink_soft_R`
  - `face_blink_both`
  - `face_focus_squint`
  - `face_smile_soft`
  - `face_mouth_open_soft`
  - `face_listen_curious`
- 阿拓 parts-v3 保留完整底层头部，并使用不切洞的 overlay patch 做眼周、口鼻和脸颊拆件，降低破面、漏光和 z-fighting 风险。

### 骨骼与动作

- 建立语义骨骼绑定，而不是依赖导出顺序。
- 所有姿态都相对 bind pose 叠加，避免动作切换后骨骼漂移。
- 将头部运动拆成 `Head / Neck / UpperChest` 的小角度分摊，降低脖子折断和脸部变形。
- 入场改为远处跑来、减速站定、朝相机招手三秒，再进入 idle。
- Idle 时双臂回到自然下垂，并按角色人格触发插兜张望、挠头、陶醉听歌、伸懒腰打哈欠。
- 聆听、思考、说话、互聊、舞蹈、旋转、高抬腿将身体动作与眨眼、眯眼、微笑、张嘴混合。
- 握手使用透明手部命中体；按住手后，肩、肘、腕根据指针/触摸高度连续跟随。
- 参考 Mesh2Motion 调整走路周期、骨盆起伏、反向摆臂、大小腿和脚掌相位。
- 漫游改为小步长、最大 45° 分段转向和限速旋转，避免突然大角度漂移。
- 呼唤回来时使用更快的跑步状态归位，相机同步恢复默认构图。

### 场景与交互

- 模型加载阶段由早期低质占位角色改为粒子轮廓；加载完成后粒子从角色周围飘散。
- 四角色、车辆和地面都加入可控投影，去除脚下装饰圆环。
- 911 使用旋转矩形碰撞体，不再只按世界坐标轴估算。
- 角色名字投影到屏幕坐标，在角色离开默认位置后随角色移动。
- 相机在空间漫游时后拉并抬升，在归位完成后回到桌面初始视角。

## 2. 2026-06-28 对另外三只角色的资产迁移

使用 Blender MCP 对 `nova.glb / muse.glb / milo.glb` 做了独立检查。原文件保留，新文件以 `*-rigged-clean-v2.glb` 输出。

| 角色 | 顶点 | 骨骼 | 修复前 Eye 主影响顶点 | clean-v2 结果 |
| --- | ---: | ---: | ---: | --- |
| 诺瓦 | 35,923 | 38 | 1,463 | 0 |
| 缪思 | 42,483 | 46 | 8,325 | 0 |
| 米洛 | 38,773 | 38 | 5,992 | 0 |

三份 clean-v2 共同通过：

- 0 个无权重顶点；
- 0 个超过四骨骼影响的顶点；
- `LeftEye / RightEye / Jaw / *_end` 主影响顶点为 0；
- 保留完整骨架；
- 七个统一 facial morph 均可被 WebGL 收集；
- 原 GLB 未被覆盖，可随时回退。

需要明确：另外三只目前仍是“完整底模 + 安全权重 + 轻量语义 morph”，尚未达到阿拓 parts-v3 的面部拆件等级。眼球、眼皮、帽檐、耳机、围裙、杯子等仍需要后续独立拓扑和刚性附件绑定。

## 3. 四 Agent 统一运行时协议

以下能力现在不再写死为阿拓专用：

- GLB 自动适配、语义骨骼绑定和 facial morph 收集；
- 入场跑步、三秒招手、人格 idle；
- 聆听、思考、对话、互聊、握手、跳舞、旋转、高抬腿；
- 20 秒无交互后错峰漫游；
- 每个角色独立步频、步幅、摇摆、速度、转向速度和路径；
- 四个名字都能随角色移动；
- 可通过“角色名 + 回来/归位”让指定角色回到自己的默认位置。

四种漫游气质：

- 阿拓：稳定、有目的，步幅适中。
- 诺瓦：稍快、动作克制，像在巡检。
- 缪思：节奏更松、身体摇摆更明显。
- 米洛：更慢、更圆润，保持松弛感。

## 4. 群体碰撞系统

碰撞解算位于 `src/webgl/crowd-motion.ts`，每一帧按以下顺序处理候选位置：

1. 限制在相机可见的舞台边界内。
2. 将角色圆形安全体推出 911 的旋转矩形安全区。
3. 将角色推出其他角色的动态圆形安全区。
4. 最多迭代四次，处理“躲开角色后又进入车辆”这类连锁碰撞。

碰撞同时用于：

- 选择下一个漫游目标；
- 每一帧实际位移；
- 漫游暂停；
- 跑回默认位置。

因此不是只保证目标点安全，移动路径中也会持续避障。路径超时后会主动选择新目标，避免两个角色面对面僵持和抖动。

## 5. 剧情式自主互动

2026-06-28 第二轮将无目的随机漫游替换为 20 秒剧情节拍。每段剧情包含“接近—停下互动—散开归位”三个阶段，并在舞台下方显示一句轻量剧情字幕：

1. 阿拓叫诺瓦一起检查路线和 911。
2. 缪思把一段像发动机节拍的新歌分享给诺瓦，诺瓦害羞点头并用脚尖打拍。
3. 米洛递出隐藏菜单点心，缪思开心弹跳回应。
4. 四位伙伴围到车前，以各自的动作完成出发确认。

剧情动作不是统一 `social` 循环，而是角色专属：

- 阿拓：指路线、张手解释、带轻微说话嘴型。
- 诺瓦：检查车辆、扫视、点头、脚尖打拍。
- 缪思：扶耳机、递出另一边耳机、律动和开心弹跳。
- 米洛：双手递点心、身体前倾、期待式微笑。

碰撞半径不再使用固定经验值，而是根据每个 GLB 归一化后的实际宽度/深度计算。911 的碰撞矩形也由加载后的真实模型长宽生成；剧情站位会先通过同一碰撞解算，再开始表演。

## 6. 当前边界与下一轮

本轮解决的是“安全、统一、可持续迭代”的底座，不代表所有角色已经达到最终动画片品质。

下一轮优先级：

1. 为诺瓦、缪思、米洛制作与阿拓 parts-v3 等价的安全拆件。
2. 独立眼球、上下眼皮、口腔、牙齿和眉部控制。
3. 帽子、耳机、杯子、围裙等附件改为刚性骨骼或 socket。
4. 用动画曲线或动作片段替代部分程序化关节摆动。
5. 增加局部路径预测，让角色提前侧身绕行，而不是进入安全半径后才被推出。
6. 增加长时间自动巡航压力测试：四角色同时漫游 10 分钟，记录最小间距、越界和卡住次数。

## 7. 验收清单

- 页面加载后，四角色按顺序跑来并招手三秒。
- Idle 双臂自然，不回到 T-pose。
- 角色头部转动时脸、帽子、耳机不被 Eye/Jaw 骨拉扯。
- 四角色均能触发七类语义表情。
- 四角色均能漫游，步态有区别。
- 任意两角色中心距离不小于两者安全半径之和。
- 任意角色不进入 911 的旋转安全矩形。
- 任意角色名字在漫游和归位时跟随。
- “诺瓦回来 / 缪思归位 / 米洛回来 / 阿拓回来”分别作用于对应角色。
- 最后一个漫游角色归位后，相机回到初始位置。
